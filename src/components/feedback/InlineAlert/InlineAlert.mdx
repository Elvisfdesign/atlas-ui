# Inline Alert

A tone-tinted message block for a form, card, or page section. Source:
Atlas's existing status tone system (`info`/`success`/`warning`/`danger`
background+text pairs, already used by Badge) applied to a larger,
icon-led message block — the inline counterpart to Banner, which spans
full page width instead.

## Overview

An icon, an optional bold title, and a description, on a tinted
background matching the chosen tone.

## Live Preview

<Canvas of={InlineAlertStories.Info} />

## Usage

Use Inline Alert for a message scoped to one section of a page — a form,
a card, a panel — not a page-wide notice (use Banner for that). `title`
is optional; omit it for a single short sentence that doesn't need a
separate lead-in.

## Sizes

One size — Inline Alert fills its container's width; height is
determined by its content.

## States

<Canvas of={InlineAlertStories.WithoutTitle} />

Inline Alert is a static display component — it has no interactive
states of its own.

## Accessibility

- `warning` and `danger` tones render `role="alert"` (assertive,
  interrupts) — `info` and `success` render `role="status"` (polite,
  announced without interrupting). Not every alert deserves the same
  urgency.
- The icon is decorative (`aria-hidden="true"`) — tone is always
  communicated by the visible title/description text as well, never by
  color or icon shape alone.

## Examples

```tsx
import { InlineAlert } from '@/components/feedback/InlineAlert';

</InlineAlert>
```
