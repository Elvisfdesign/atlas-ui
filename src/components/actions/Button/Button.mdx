# Button

The primary way a person triggers an action in Atlas. Source:
Atlas UI System, Actions → Action / Button (Type × Size × State, 40 variants).

## Overview

Four variants (primary, secondary, ghost, destructive) and two sizes cover
every action-button need in Atlas Product today — there is no fifth
variant or third size waiting to be discovered, so reach for one of these
before inventing a one-off style.

## Usage

Use **primary** for the one main action in a view (Approve, Save, Continue)
— there should only be one primary button visible at a time. Use
**secondary** for the alternative action next to it (Cancel, Back). Use
**ghost** for low-emphasis actions inside dense UI (a toolbar, a table row)
where a bordered button would add visual weight the context doesn't need.
Use **destructive** only for actions that delete or irreversibly change
data — never for anything else, so its color stays meaningful.

## Sizes

Two sizes: **medium** (36px tall, the default) and **small** (29px, for
dense contexts like table row actions or toolbars). There is no `large` —
Atlas Product doesn't use one, so this library doesn't invent one.

`loading` isn't a state documented in the Atlas Figma files — it's added
here because a production button needs one, built entirely from existing
tokens (the same spinner color as the label, no new color introduced).
While `loading` is true, the button is disabled and marked `aria-busy`.

## Accessibility

- Renders a real `<button type="button">` — full keyboard operability
  (Tab to focus, Enter/Space to activate) comes from the browser, not
  custom key handling.
- Focus is always visible: a 2px `border-focus` outline, never suppressed.
- `disabled` and `loading` both set the native `disabled` attribute, so
  assistive tech and keyboard navigation skip the button the same way.
  `loading` additionally sets `aria-busy="true"`.
- The spinner icon is `aria-hidden` — the button's accessible name stays
  its text content, not "loading spinner approve."

## Examples

```tsx
import { Button } from '@/components/actions/Button';
import { Check } from 'lucide-react';

function Example() {
  return (
    <Button variant="primary" size="medium" icon={<Check size={16} />}>
      Approve &amp; Continue
    </Button>
  );
}
```

```tsx
// Async action with a loading state
function ApproveButton({ onApprove }: { onApprove: () => Promise<void> }) {
  const [loading, setLoading] = useState(false);
  return (
    <Button
      loading={loading}
      onClick={async () => {
        setLoading(true);
        await onApprove();
        setLoading(false);
      }}
    >
      Approve
    </Button>
  );
}
```
