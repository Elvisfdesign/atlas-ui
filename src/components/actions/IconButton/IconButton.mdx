# Icon Button

A single, chromeless button for icon-only actions — notifications, search,
close, overflow menus. Source: Atlas UI System, Actions → Action / Icon
Button (Size × State).

## Overview

One visual treatment, two sizes, and a required `aria-label` — Icon Button
is deliberately the smaller, simpler sibling of Button, for places where an
icon alone is enough to say what a control does.

## Usage

Reach for Icon Button when the action is unambiguous from its icon alone
in context — a close (✕) in a modal header, a bell for notifications in a
topbar. If there's any chance the icon alone is ambiguous, use a regular
Button with a visible label instead; don't rely on a tooltip to explain an
Icon Button that shouldn't need one.

## Variants

There is one visual treatment — chromeless, tinted on hover/press — that
covers every documented Figma state. Icon Button doesn't have a
`variant` prop; if a future use case genuinely needs a bordered or filled
icon button, that's a new variant to add deliberately, not a reason to
overload this one.

## Sizes

Medium (36px) is the default; Small (28px) matches dense contexts like a
table's row-action column.

## Accessibility

- **`aria-label` is a required prop, not optional** — TypeScript enforces
  it. An icon-only button with no accessible name is a common, serious
  accessibility failure; this component makes it impossible to ship one.
- The icon itself is always rendered with `aria-hidden="true"`, wrapped in
  its own `<span>`, so screen readers announce only the button's
  `aria-label`.
- Fully keyboard operable as a native `<button>`.
