# Notification Panel

The bell-icon panel present in Atlas Product's topbar on every screen —
a 36px chip with a small corner dot indicating unread notifications.
Composes Popover for the floating panel rather than a custom
implementation.

## Overview

A bell trigger (with an optional unread dot) that opens a scrollable
list of notifications, each with a title, optional description, and a
relative time.

## Live Preview

<Canvas of={NotificationPanelStories.Default} />

## Usage

Pass `items` and set `hasUnread` when any of them are new — this drives
both the trigger's corner dot and its accessible name. Unread items
(`read: false`) get a tinted background so they're scannable at a
glance without relying on the corner dot alone.

<Canvas of={NotificationPanelStories.Unread} />

## Sizes

One size (320px panel width, capped at 320px scrollable height for the
list).

## States

<Canvas of={NotificationPanelStories.Empty} />

## Accessibility

- The trigger's accessible name changes between "Notifications" and
  "Notifications (unread)" depending on `hasUnread` — the unread state
  is never conveyed by the corner dot alone.
- Each notification is a real `<button>`, so it's reachable and
  activatable by keyboard, not a `<div>` with a click handler.

## Examples

```tsx
import { NotificationPanel } from '@/components/overlays/NotificationPanel';

<NotificationPanel
  hasUnread
  items={[
    { id: '1', title: 'Workflow published', description: 'Invoice Approval is now live.', time: '2m ago', read: false },
    { id: '2', title: 'Weekly digest ready', time: 'Yesterday', read: true },
  ]}
  onItemClick={(id) => markRead(id)}
/>
```
