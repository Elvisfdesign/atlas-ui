# Tabs

Switches between views that share the same context. Source: Atlas
Product ÔÇö the Review Queue's filter tabs (All 12, Invoices 5, Contracts
4, Receipts 3 ÔÇö each a label + count, active tab underlined in accent).

## Overview

Built on [Radix's Tabs primitive](https://www.radix-ui.com/primitives/docs/components/tabs)
rather than hand-rolled, so roving-tabindex arrow-key navigation and
`Home`/`End` support are solved once, correctly ÔÇö the same rationale
already used for Tooltip.

## Usage

Use Tabs to divide one dataset into overlapping subsets the person
switches between without leaving the page (all documents vs. just
invoices). Don't use Tabs for a multi-step sequence ÔÇö that's Step
Navigation's job, not this component's.

## Anatomy

`Tabs` (root, holds selection state) Ôćĺ `TabsList` (the row, with the
bottom divider) Ôćĺ `Tab` (one trigger, with an optional `count`) Ôćĺ
`TabPanel` (the content shown for the selected tab).

## Variants

One visual treatment ÔÇö Tabs doesn't have a `variant` prop.

## Sizes

One size (34px tall triggers), matching Atlas Product exactly.

## States

The active tab gets `text-primary`, a bold weight, and a 2px accent
underline; inactive tabs are `text-tertiary`, going to `text-primary` on
hover.

## Accessibility

- Renders proper `role="tablist"`/`role="tab"`/`role="tabpanel"` with
  `aria-selected` and `aria-controls`/`aria-labelledby` wiring, all from
  Radix.
- Arrow keys move focus and selection between tabs; only the active
  tab is in the Tab order (roving tabindex) ÔÇö matching the WAI-ARIA Tabs
  pattern rather than putting every tab in the Tab sequence.

## Examples

```tsx
import { Tabs, TabsList, Tab, TabPanel } from '@/components/navigation/Tabs';

<Tabs value={filter} onValueChange={setFilter}>
  <TabsList>
    <Tab value="all" count={12}>All</Tab>
    <Tab value="invoices" count={5}>Invoices</Tab>
  </TabsList>
  <TabPanel value="all">{/* ... */}</TabPanel>
  <TabPanel value="invoices">{/* ... */}</TabPanel>
</Tabs>
```
