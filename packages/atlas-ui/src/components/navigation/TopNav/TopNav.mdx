<DocsTitle>Top Navigation</DocsTitle>

<DocsIntro>
The app shell's top bar. Source: Atlas Product — the Review Queue topbar
(128px content row + a 1px bottom divider, 32px horizontal padding,
global search on the left, notifications and the user menu on the
right).
</DocsIntro>

<DocsSection title="Overview">

A structural shell only. Every piece Atlas Product places inside it —
the search field, the notification trigger, the user avatar — is already
its own component (Search Input, Icon Button, Avatar). `TopNav` lays
them out; it doesn't reimplement any of them.

</DocsSection>

<DocsSection title="Live Preview">

<Canvas of={TopNavStories.Default} />

</DocsSection>

<DocsSection title="Usage">

Compose `TopNav` with a `SearchInput` on the left and whatever page-level
actions (notifications, user menu) belong on the right, using ordinary
flex children — `TopNav` doesn't take a fixed set of slot props, since
what belongs in the topbar varies by page.

</DocsSection>

<DocsSection title="Variants">

One visual treatment — `TopNav` doesn't have a `variant` prop.

</DocsSection>

<DocsSection title="Sizes">

One size (128px content height + 1px divider), matching Atlas Product
exactly.

</DocsSection>

<DocsSection title="States">

Stateless — `TopNav` is a layout container. Its children (Search Input,
Icon Button, Avatar) carry their own states.

</DocsSection>

<DocsSection title="Accessibility">

Renders as a `<header>` — the page's banner landmark, standard for a
top-level application bar. Its children remain independently keyboard
operable through their own implementations.

</DocsSection>

<DocsSection title="Examples">

```tsx
import { TopNav } from '@/components/navigation/TopNav';
import { SearchInput } from '@/components/forms/SearchInput';

    <IconButton icon={<Bell />} aria-label="Notifications" />
    <Avatar name="Elvis Meraz" color="inverse" />
  </div>
</TopNav>
```
