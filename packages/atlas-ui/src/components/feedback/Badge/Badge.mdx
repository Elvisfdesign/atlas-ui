# Badge

A small, pill-shaped label communicating status. Source: Atlas UI System,

## Overview

Nine tones cover every status Atlas Product currently shows â€” general
states (neutral, info, success, warning, danger) and workflow-specific
ones (processing, review, draft, ai) â€” in a single, non-interactive shape.

## Usage

Status is always communicated by the label's **text** and its **color**
together â€” never color alone. Don't ship a Badge with an ambiguous or
empty label just to show a color. Pick the tone that matches the status's
real meaning (`danger` for a failure, not just because red looks urgent).

`processing` and `review` intentionally share visual treatment with `info`
and `warning` respectively â€” they exist as distinct tone names because the
*meaning* differs even when the color doesn't, which matters for anyone
reading the component's usage in code.

## Sizes

One size. Atlas Product doesn't show a Badge at more than one size, so
this library doesn't invent one.

## States

Badge is a static, non-interactive label â€” it has no hover, focus, or
disabled state.

## Accessibility

- Renders as a `<span>` with the tone communicated through both text and
  color, so the status is available to screen readers and to anyone with
  color-vision deficiency.
- Not focusable and not a control â€” if a badge needs to be clickable
  (e.g. to filter by status), wrap it in a real interactive element
  rather than adding click handling to the Badge itself.
