<DocsLayout>

<DocsTitle>Loading State</DocsTitle>

<DocsIntro>
A centered spinner + label for a section or card body that's still
loading. Distinct from Skeleton, which mimics the shape of the content
that's coming rather than showing a spinner.
</DocsIntro>

<DocsSection title="Overview">

An animated spinner icon and a text label, centered in their container.

</DocsSection>

<DocsSection title="Live Preview">

<Canvas of={LoadingStateStories.Default} />

</DocsSection>

<DocsSection title="Usage">

Use Loading State for a page section, card, or panel whose final layout
isn't known yet, or where a shape-matching Skeleton isn't worth building.
Prefer Skeleton when the eventual content's shape is well known (a
table, a list, a card) — it reduces layout shift and feels faster.

</DocsSection>

<DocsSection title="Variants">

<Canvas of={LoadingStateStories.CustomLabel} />

</DocsSection>

<DocsSection title="Sizes">

<Canvas of={LoadingStateStories.Small} />

`sm` (20px spinner) suits a smaller card or inline section; `md` (28px,
default) suits a full page-section placeholder.

</DocsSection>

<DocsSection title="States">

Loading State has one state — it's already the loading state itself.

</DocsSection>

<DocsSection title="Accessibility">

- Renders `role="status"`, so assistive technology announces the label
  when the loading section mounts, without interrupting other content.
- The spinner icon is decorative (`aria-hidden="true"`) — the label is
  the only thing that needs to be read.

</DocsSection>
