# Toast

A transient, corner-anchored notification. Source: no dedicated Product
frame — toasts are transient by nature and don't sit in a static mockup
— built on Radix's Toast primitive, which owns auto-dismiss timing,
swipe-to-dismiss, and pause-on-hover/focus. Styled with the same
surface-elevated + elevation-high recipe already used for Line Chart's
tooltip, and shares its tone icon with Inline Alert and Banner.

## Overview

An icon, a title, an optional description, and a close button, rendered
into a fixed viewport stack.

## Live Preview

## Usage

Render `ToastProvider` once near your app root, and `ToastViewport` once
where you want the stack to anchor (bottom-right by default). Render a
`Toast` for each notification your app queues — Atlas doesn't ship a
global toast-queue store, so the calling app manages which toasts are
currently mounted (an array in state is the common pattern), the same
way it would with the raw Radix primitive.

## Sizes

One size — the viewport caps toasts at a max width (384px) so they never
grow to fill an entire wide screen.

## Accessibility

- Radix's Toast primitive announces its content via an `aria-live`
  region, handles focus so a toast never steals it from what the user
  was doing, and supports Escape-to-dismiss and swipe gestures.
- The close button is a real, labeled `IconButton` ("Dismiss
  notification").

## Examples

```tsx
import { Toast, ToastProvider, ToastViewport } from '@/components/feedback/Toast';

function App() {
  const [toasts, setToasts] = useState<{ id: string; title: string }[]>([]);
  return (
    <ToastProvider>
      {/* ...app... */}
      {toasts.map((t) => (
        <Toast key={t.id} tone="success" title={t.title} onOpenChange={(open) => !open && dismiss(t.id)} />
      ))}
      <ToastViewport />
    </ToastProvider>
  );
}
```
