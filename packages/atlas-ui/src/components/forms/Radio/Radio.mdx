<DocsTitle>Radio</DocsTitle>

<DocsIntro>
One choice from a small, always-visible set of options.
</DocsIntro>

<DocsSection title="Overview">

Not yet its own componentized node in the Atlas UI System (see the React
Readiness matrix) — built as a circular sibling of Checkbox, reusing the
same border/accent/focus treatment so the two read as one family of
selection control. `Radio` reads its shared `name` and selected value
from a wrapping `RadioGroup`.

</DocsSection>

<DocsSection title="Live Preview">

<Canvas of={RadioStories.Group} />

</DocsSection>

<DocsSection title="Usage">

Use Radio (in a `RadioGroup`) when there are 2–5 mutually exclusive
options and showing them all at once helps the person decide. For a
longer list, or when screen space is tight, use Select instead.

</DocsSection>

<DocsSection title="Variants">

One visual treatment — Radio doesn't have a `variant` prop.

</DocsSection>

<DocsSection title="Sizes">

One size (17px), matching Checkbox exactly.

</DocsSection>

<DocsSection title="States">

<Canvas of={RadioStories.Disabled} />

</DocsSection>

<DocsSection title="Accessibility">

- `RadioGroup` renders `role="radiogroup"` with an `aria-label`, so
  assistive tech announces the set as one group, not unrelated controls.
- Arrow keys move focus (and selection) between options, and Tab moves
  into/out of the whole group — standard native radio-group keyboard
  behavior, since each `Radio` is a real `<input type="radio">` sharing
  one `name`.
- Each option's label is a real `<label htmlFor>`.

</DocsSection>
