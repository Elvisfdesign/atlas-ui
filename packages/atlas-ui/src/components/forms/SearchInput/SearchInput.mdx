# Search Input

The global search field in Atlas Product's topbar. Source: Atlas
Product — the Review Queue topbar (320×36, leading Search icon,
"Search anything..." placeholder, trailing ⌘K shortcut chip).

## Overview

A distinct component from Input rather than an `icon` prop on it —
Search Input carries `type="search"` semantics and an optional shortcut
chip that a generic text field shouldn't.

## Live Preview

## Usage

Use Search Input for the one global, app-wide search field in the topbar.
For search *within* a specific view (filtering a table, say), the same
component works without the `shortcut` prop — the chip is specifically
for a search that has a real keyboard shortcut bound to it.

## Variants

<Canvas of={SearchInputStories.Default} />

Without `shortcut`, Search Input is a plain icon + placeholder field —
appropriate for any in-page search that isn't bound to a global hotkey.

## Sizes

One size (36px), matching the Atlas Product topbar exactly.

## Accessibility

- Rendered with `role="searchbox"` and `type="search"`, so assistive
  technology announces it as a search field, not a generic text box.
- The Search icon and shortcut chip are both `aria-hidden` — the field's
  accessible name comes from an explicit `aria-label` you provide (it has
  no visible `<label>` by design, matching the topbar's compact layout).
