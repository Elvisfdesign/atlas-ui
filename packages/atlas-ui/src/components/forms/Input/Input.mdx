<DocsTitle>Input</DocsTitle>

<DocsIntro>
The default single-line text field in Atlas. Source: Atlas UI System,
Forms → Form / Text Input (a 6-state variant set: default, hover, focus,
filled, disabled, error).
</DocsIntro>

<DocsSection title="Overview">

Input covers every plain text-entry need — names, IDs, single-line
values. It isn't a `state` prop with six options: hover and focus come
from the browser for free, and `filled` is just what the field looks like
once it has a value. Only `error` and `disabled` are things a caller sets.

</DocsSection>

<DocsSection title="Usage">

Always pass a `label` rather than relying on `placeholder` alone —
placeholder text disappears the moment someone starts typing, so it can't
carry the field's identity by itself. Use `helperText` for guidance that
should always be visible, and `error` only once a real validation problem
exists.

</DocsSection>

<DocsSection title="Variants">

<Canvas of={InputStories.WithLeadingIcon} />

A leading icon is the one supported visual variant — for a unit, a
currency symbol, or context like a mail icon on an email field. It's
`aria-hidden`, same as everywhere else in Atlas.

</DocsSection>

<DocsSection title="Sizes">

<Canvas of={InputStories.Small} />

**medium** (36px, the default) matches Button's medium height, so an Input
and a Button sit flush in the same row. **small** (29px) is for dense
layouts — filter bars, inline table editing.

</DocsSection>

<DocsSection title="Accessibility">

- The label is a real `<label htmlFor>`, not a placeholder or a floating
  span — clicking it focuses the input, and screen readers always
  announce it.
- `error` sets `aria-invalid` and points `aria-describedby` at the error
  text, which renders with `role="alert"` so it's announced as soon as it
  appears.
- Focus is always visible: a 2px accent ring, matching every other Atlas
  interactive element.

</DocsSection>
