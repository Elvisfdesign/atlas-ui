<Meta of={ActivityFeedStories} />


<DocsLayout>

<DocsTitle>Activity Feed</DocsTitle>

<DocsIntro>
A recent-activity list of processed documents. Source: Atlas Product's
Analytics "Recent Activity" card — a header with a "View all" link,
column headers (DOCUMENT/TYPE/STATUS/TIME), and rows composing a
document icon + name, a type label, a status Badge, and a relative time.
</DocsIntro>

<DocsSection title="Overview">

Composes `ChartCard` for its title/action header, `FileRow` for the
document cell, and `Badge` for status — rather than reimplementing any
of their layout.

</DocsSection>

<DocsSection title="Live Preview">

<Canvas of={ActivityFeedStories.Default} />

</DocsSection>

<DocsSection title="Usage">

Pass `items` (each with a `fileName`, `type`, `status`, and `time`).
`statusTone` defaults from the status text itself ("Completed" →
success, "Failed" → danger, "Review" → warning, "Processing" →
processing) — override it only for a status outside that set.

</DocsSection>

<DocsSection title="Variants">

<Canvas of={ActivityFeedStories.WithViewAll} />

Pass `onViewAll` to show the "View all" link in the header, matching the
Analytics card exactly; omit it for a feed with no link to a fuller
list.

</DocsSection>

<DocsSection title="Sizes">

One size — the feed fills its container's width; row height and column
widths (Type/Status/Time are fixed-width, Document flexes) are fixed to
match the Analytics pattern.

</DocsSection>

<DocsSection title="States">

<Canvas of={ActivityFeedStories.Empty} />

</DocsSection>

<DocsSection title="Accessibility">

- The "View all" affordance renders as a real `<button>` with visible
  focus styling, not a styled `<span>` with a click handler.
- Status is always a Badge with visible text — never conveyed by row
  color alone.
- Column headers use the same uppercase/tracked treatment as Table's
  header cells, for a consistent "this is a header row" signal across
  Atlas even outside a literal `<table>`.

</DocsSection>

<DocsSection title="Examples">

```tsx
import { ActivityFeed } from '@/components/data-display/ActivityFeed';

<ActivityFeed
  onViewAll={() => navigate('/activity')}
  items={[
```
