<DocsSection title="Overview">

A single headline number, its trend, and a stated comparison period — the
building block of every Atlas Product dashboard summary row.

</DocsSection>

<DocsSection title="Live Preview">

<Canvas of={KPICardStories.Positive} />

</DocsSection>

<DocsSection title="Usage">

One KPI Card per metric — label, value, and a trend comparison against a
stated period ("vs last 7 days"). Always include the comparison subtext;
a bare number with an up/down arrow and no stated baseline isn't
meaningful. Lay multiple cards out in a row (see the `Row` story) rather
than stacking them, matching how Atlas Product's dashboards use them.

</DocsSection>

<DocsSection title="Variants">

<Canvas of={KPICardStories.Negative} />

`trend` drives both the arrow direction and the color (success green for
positive, danger red for negative) — there's no way to show a positive
number in red or vice versa, since that would work against the color's
meaning elsewhere in Atlas.

</DocsSection>

<DocsSection title="Sizes">

One size (260×130px). Atlas Product doesn't show KPI Card at another size.

</DocsSection>

<DocsSection title="States">

<Canvas of={KPICardStories.WithMenu} />

`showMenu` reveals an overflow ("more options") affordance in the
top-right corner — off by default, since not every KPI Card needs one.

<Canvas of={KPICardStories.Row} />

</DocsSection>

<DocsSection title="Accessibility">

- Structured as plain text content (label, value, trend, subtext) in
  reading order — nothing depends on positioning or color alone to convey
  meaning; the trend direction is stated in the number itself (`+12.5%`
  vs `-4.3%`), not just implied by an icon.
- The overflow button (`showMenu`) has an accessible name derived from the
  card's own label ("More options for TOTAL WORKFLOWS"), so multiple KPI
  Cards on one page remain distinguishable to screen reader users.

</DocsSection>

<DocsSection title="Examples">

```tsx
import { KPICard } from '@/components/data-display/KPICard';

<div className="flex gap-4">
  <KPICard
    label="TOTAL WORKFLOWS"
    value={128}
    trend="positive"
    trendValue="+12.5%"
    subtext="vs last 7 days"
  />
  <KPICard
    label="DOCUMENTS PROCESSED"
    value={342}
    trend="negative"
    trendValue="-4.3%"
    subtext="vs last 7 days"
    showMenu
    onMenuClick={() => openCardMenu('documents')}
  />
</div>
```

Note: the Figma component's metric text renders at 26px, while the named
`Numeric Metric` type-scale token in Foundations documents 28px — a small,
pre-existing drift between the token and its most visible usage. KPI Card
matches the shipped component (26px), not the token, since the component
is the literal, shipped artifact.
