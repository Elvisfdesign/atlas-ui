# Workflow Card

A single automated workflow's summary card. Source: Atlas Product's
Workflow Builder pipeline cards — a title and status Badge, a one-line
description, and a metrics row (a volume count and a success percentage
separated by "·").

## Overview

Composes the existing `Badge` for status rather than reimplementing a
status pill, and renders as a `<button>` when `onClick` is provided or a
plain `<div>` otherwise — the same card used for a static summary list
or a clickable workflow-selection grid.

## Live Preview

<Canvas of={WorkflowCardStories.Active} />

## Usage

Provide `title`, `description`, and `status`; `status` drives a
reasonable default Badge tone ("Active" → success, "Draft" → draft,
"Scheduled" → info, "Archived" → neutral), overridable via
`statusTone`. Include `volume`/`successRate` for workflows with
activity to report — omit both for a workflow that hasn't run yet.

## Sizes

One size (280px in this preview, but the card fills its container width
— lay multiple cards out in a responsive grid).

## States

<Canvas of={WorkflowCardStories.Clickable} />

Passing `onClick` renders the card as a real `<button>` with a hover
border change, for use in a selectable workflow grid; omitting it
renders a plain, non-interactive `<div>` for a read-only summary list.

## Accessibility

- Renders as a semantic `<button>` (not a `<div>` with a click handler)
  whenever it's interactive, so it's reachable and activatable by
  keyboard automatically.
- Status is always shown as Badge text, never color alone.

## Examples

```tsx
import { WorkflowCard } from '@/components/data-display/WorkflowCard';

<WorkflowCard
  title="Invoice Approval"
  description="Routes vendor invoices over $5,000 to finance for manual sign-off."
  status="Active"
  volume="1,245"
  successRate="98.2%"
  onClick={() => openWorkflow('invoice-approval')}
/>
```
