<DocsLayout>

<DocsTitle>User Row</DocsTitle>

<DocsIntro>
A person's avatar, name, and an optional secondary line. Source: Atlas
Product — the Review Queue table's assignee cell.
</DocsIntro>

<DocsSection title="Overview">

Composes the `Avatar` primitive directly (24px `small` by default,
matching the assignee chip) rather than reimplementing photo/initials/
fallback logic, paired with the person's name and an optional meta line.

</DocsSection>

<DocsSection title="Live Preview">

<Canvas of={UserRowStories.Default} />

</DocsSection>

<DocsSection title="Usage">

Use in a table cell or list wherever a row needs to identify a specific
person — an assignee, a reviewer, an author. Pass `avatarSrc` for a
photo; it falls back to initials automatically (inherited from Avatar)
if the image fails to load.

</DocsSection>

<DocsSection title="Variants">

<Canvas of={UserRowStories.List} />

</DocsSection>

<DocsSection title="Sizes">

<Canvas of={UserRowStories.Medium} />

`small` (default, 24px avatar) matches the Review Queue assignee cell;
`medium` (34px avatar) reads better in a more spacious context.

</DocsSection>

<DocsSection title="States">

<Canvas of={UserRowStories.WithMeta} />

User Row is a static display component — it has no interactive states of
its own. Long names truncate with an ellipsis.

</DocsSection>

<DocsSection title="Accessibility">

- The underlying `Avatar` sets `role="img"` with an accessible name
  equal to the person's name, so the visual chip and the visible name
  text are consistent for screen reader users, not redundant or
  conflicting.
- Name and meta render as plain text in reading order.

</DocsSection>
