<DocsLayout>

<DocsTitle>Detail List</DocsTitle>

<DocsIntro>
A plain label/value row list. Source: the label-above-value text stack
used in every Document Review "Extracted Fields" row (12px tertiary
label, 13px primary value), decomposed here into its own primitive
without the confidence percentage and AI icon chip — those belong
specifically to the (AI) Extracted Field Row, which composes this same
text-stack pattern and adds a Confidence Indicator on top.
</DocsIntro>

<DocsSection title="Overview">

A vertical list of label/value pairs, each separated by a hairline
divider, rendered as a semantic `<dl>`/`<dt>`/`<dd>` structure.

</DocsSection>

<DocsSection title="Live Preview">

<Canvas of={DetailListStories.Default} />

</DocsSection>

<DocsSection title="Usage">

Use for any plain metadata display — document properties, account
details, a settings summary — anywhere a value doesn't need a paired
confidence score. If a value does need one, use the (AI) Extracted
Field Row instead, not this component with a confidence badge bolted on
separately.

</DocsSection>

<DocsSection title="Variants">

<Canvas of={DetailListStories.SingleItem} />

`value` accepts any `ReactNode`, so a value can be plain text, a Badge,
or any other inline content — not just a string.

</DocsSection>

<DocsSection title="Sizes">

One size — Detail List fills its container's width; row height is
determined by its content.

</DocsSection>

<DocsSection title="States">

Detail List is a static display component — it has no interactive or
loading states of its own.

</DocsSection>

<DocsSection title="Accessibility">

- Renders as a real `<dl>`/`<dt>`/`<dd>` list, giving assistive
  technology explicit label/value semantics rather than plain, unrelated
  text nodes.

</DocsSection>
