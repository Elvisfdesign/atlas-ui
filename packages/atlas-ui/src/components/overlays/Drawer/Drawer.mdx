# Drawer

An edge-anchored modal panel. Built on the same Radix Dialog primitive
as Modal — full focus trapping and Escape-to-close — but anchored to a
screen edge instead of centered, for content that needs more room than
Modal's 448px (a document's full detail view, a multi-field editor).

## Overview

`Drawer` (root), `DrawerTrigger`, `DrawerContent` (with a `side` prop),
and `DrawerTitle`/`DrawerDescription`/`DrawerHeader`/`DrawerBody`/
`DrawerFooter` slot components — the same shape as Modal's.

## Live Preview

<Canvas of={DrawerStories.Right} />

## Usage

Use Drawer instead of Modal when the content is naturally a detail view
or a longer form — `DrawerBody` scrolls independently of the header/
footer, so a tall Drawer never breaks its own layout.

## Variants

<Canvas of={DrawerStories.Left} />

`side="right"` (default) or `side="left"` — pick based on where the
triggering context sits (e.g. a table row's detail view opens from the
right).

## Sizes

One size (max-width 384px / `max-w-sm`), full viewport height.

## States

Open/closed is the only state, shown via the trigger interaction in the
Live Preview.

## Accessibility

- Same accessibility guarantees as Modal: focus trap, focus return,
  Escape-to-close, background scroll lock.
- `DrawerTitle` becomes the dialog's accessible name automatically.

## Examples

```tsx
import { Button } from '@/components/actions/Button';
import {
  Drawer,
  DrawerBody,
  DrawerContent,
  DrawerHeader,
  DrawerTitle,
  DrawerTrigger,
} from '@/components/overlays/Drawer';

<Drawer>
  <DrawerTrigger asChild>
    <Button variant="secondary">View details</Button>
  </DrawerTrigger>
  <DrawerContent side="right">
    <DrawerHeader>
      <DrawerTitle>Invoice_9821.pdf</DrawerTitle>
    </DrawerHeader>
    <DrawerBody>{/* detail content */}</DrawerBody>
  </DrawerContent>
</Drawer>
```
