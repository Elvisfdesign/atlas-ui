<DocsTitle>Dropdown Menu</DocsTitle>

<DocsIntro>
The floating panel Atlas's overflow ("more options") affordances open
into. Source: no single dedicated Product frame — this is the panel
behind existing overflow triggers like KPI Card's `showMenu` and any
`MoreHorizontal` Icon Button. Built on Radix's Dropdown Menu primitive,
which owns keyboard navigation (arrow keys, type-ahead), Escape-to-
close, and focus return to the trigger.
</DocsIntro>

<DocsSection title="Overview">

`DropdownMenu` (the root), `DropdownMenuTrigger`, `DropdownMenuContent`,
and `DropdownMenuItem`/`DropdownMenuSeparator`/`DropdownMenuLabel` —
composable primitives matching plain HTML menu structure, sharing their
panel styling with Context Menu, User Menu, and Command Menu.

</DocsSection>

<DocsSection title="Live Preview">

<Canvas of={DropdownMenuStories.Default} />

</DocsSection>

<DocsSection title="Usage">

Wrap any trigger element in `DropdownMenuTrigger asChild` (an `IconButton`
with a `MoreHorizontal` icon is the most common case), and list
`DropdownMenuItem`s inside `DropdownMenuContent`. Pass `icon` and
`shortcut` props on an item for a leading icon and trailing keyboard
hint; pass `destructive` to color an item (like "Delete") with the
danger tone.

</DocsSection>

<DocsSection title="Variants">

<Canvas of={DropdownMenuStories.WithLabelAndDisabledItem} />

`DropdownMenuLabel` adds a small uppercase section heading above a
group of items; pass `disabled` on any item to make it inert without
removing it from the list.

</DocsSection>

<DocsSection title="Sizes">

One size — the menu's width is determined by its content (with a
180px minimum), not a fixed scale.

</DocsSection>

<DocsSection title="States">

Hover/keyboard-highlighted, disabled, and destructive states are shown
in the Live Preview and Variants above.

</DocsSection>

<DocsSection title="Accessibility">

- Radix wires full `menu`/`menuitem` ARIA roles, roving-tabindex arrow-
  key navigation, type-ahead, and Escape-to-close automatically.
- Focus returns to the trigger element when the menu closes, so keyboard
  users never lose their place.
- A disabled item is excluded from keyboard navigation and marked
  `data-disabled`, not just visually dimmed.

</DocsSection>

<DocsSection title="Examples">

```tsx
import { Pencil, Trash2 } from 'lucide-react';
import { IconButton } from '@/components/actions/IconButton';
import {
  DropdownMenu,
  DropdownMenuContent,
  DropdownMenuItem,
  DropdownMenuSeparator,
  DropdownMenuTrigger,
} from '@/components/overlays/DropdownMenu';

<DropdownMenu>
  <DropdownMenuTrigger asChild>
    <IconButton icon={<MoreHorizontal />} aria-label="More options" />
  </DropdownMenuTrigger>
  <DropdownMenuContent>
    <DropdownMenuItem icon={<Pencil size={14} />}>Rename</DropdownMenuItem>
    <DropdownMenuSeparator />
    <DropdownMenuItem icon={<Trash2 size={14} />} destructive>Delete</DropdownMenuItem>
  </DropdownMenuContent>
</DropdownMenu>
```
