<DocsTitle>Command Menu</DocsTitle>

<DocsIntro>
A searchable Cmd+K command palette. Source: Atlas Product — every
screen's topbar search input shows a "⌘K" chip, the real, recurring
trigger for exactly this kind of overlay. Built on `cmdk` (the standard
companion to Radix for this pattern; it wraps Radix's Dialog internally,
so Escape-to-close, focus trapping, and portaling are already handled),
styled to match Atlas's other floating panels.
</DocsIntro>

<DocsSection title="Overview">

`CommandMenu` (the dialog + search engine), `CommandMenuInput`,
`CommandMenuList`, `CommandMenuEmpty`, `CommandMenuGroup`,
`CommandMenuItem`, and `CommandMenuSeparator`.

</DocsSection>

<DocsSection title="Live Preview">

<Canvas of={CommandMenuStories.Default} />

</DocsSection>

<DocsSection title="Usage">

Bind `CommandMenu` to a global keyboard shortcut (⌘K/Ctrl+K) in your
app shell, controlling `open`/`onOpenChange` from that state. Group
related items with `CommandMenuGroup`'s `heading`, and always include a
`CommandMenuEmpty` so a no-results search state isn't a blank panel.

</DocsSection>

<DocsSection title="Variants">

Command Menu has one visual treatment — grouping and separators are
optional based on how many commands you're listing.

</DocsSection>

<DocsSection title="Sizes">

One size (max-width 512px), with the results list capped at a scrollable
320px so a long list of matches never grows unbounded.

</DocsSection>

<DocsSection title="States">

Typing in the input live-filters the list (shown in the Live Preview);
the `CommandMenuEmpty` message appears automatically when nothing
matches.

</DocsSection>

<DocsSection title="Accessibility">

- Inherits Radix Dialog's focus trap, Escape-to-close, and portal
  behavior via `cmdk`'s built-in wrapper.
- `label` (required on `CommandMenu`) provides the dialog's accessible
  name — it isn't shown visibly, since the visible input placeholder
  already communicates purpose to sighted users.
- Full keyboard navigation (arrow keys, Enter to select) is built in,
  including optional vim-style bindings.

</DocsSection>

<DocsSection title="Examples">

```tsx
import { useState, useEffect } from 'react';
import { Home, Workflow } from 'lucide-react';
import {
  CommandMenu,
  CommandMenuEmpty,
  CommandMenuGroup,
  CommandMenuInput,
  CommandMenuItem,
  CommandMenuList,
} from '@/components/overlays/CommandMenu';

function AppCommandMenu() {
  const [open, setOpen] = useState(false);
  useEffect(() => {
    const handler = (e: KeyboardEvent) => {
      if (e.key === 'k' && (e.metaKey || e.ctrlKey)) {
        e.preventDefault();
        setOpen((o) => !o);
      }
    };
    document.addEventListener('keydown', handler);
    return () => document.removeEventListener('keydown', handler);
  }, []);

  return (
    <CommandMenu open={open} onOpenChange={setOpen} label="Search commands">
      <CommandMenuInput placeholder="Search anything..." />
      <CommandMenuList>
        <CommandMenuEmpty>No results found.</CommandMenuEmpty>
        <CommandMenuGroup heading="Navigate">
          <CommandMenuItem onSelect={() => navigate('/')}>
            <Home size={14} aria-hidden="true" />
            Go to Dashboard
          </CommandMenuItem>
          <CommandMenuItem onSelect={() => navigate('/workflows')}>
            <Workflow size={14} aria-hidden="true" />
            Go to Workflows
          </CommandMenuItem>
        </CommandMenuGroup>
      </CommandMenuList>
    </CommandMenu>
  );
}
```
