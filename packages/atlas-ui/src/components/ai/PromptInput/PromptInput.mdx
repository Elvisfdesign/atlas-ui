<DocsSection title="Overview">

The composer at the bottom of Assistant Panel — a text field and a
send button, submitting on Enter or a click.

</DocsSection>

<DocsSection title="Live Preview">

<Canvas of={PromptInputStories.Default} />

</DocsSection>

<DocsSection title="Usage">

Prompt Input is uncontrolled by default — read the submitted value from
`onSubmit` rather than tracking `value` yourself, unless you need to
control it (e.g. to clear the field after sending).

</DocsSection>

<DocsSection title="Variants">

Layout and styling are fixed to match the grounded Product pattern.

</DocsSection>

<DocsSection title="Sizes">

One size — a full-width single-line composer.

</DocsSection>

<DocsSection title="States">

<Canvas of={PromptInputStories.Disabled} />

</DocsSection>

<DocsSection title="Accessibility">

- The field is a real `<input>`; the send button has an explicit
  `aria-label="Send"` since it carries only an icon.
- Submitting via Enter and via the send button both fire the same
  `onSubmit` handler (a native `<form>` under the hood).

</DocsSection>
