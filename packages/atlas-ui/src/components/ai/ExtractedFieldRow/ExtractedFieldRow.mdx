# Extracted Field Row

Source: Atlas Product — Document Review's Extracted Fields card
(`field-row`): a label + value stack on the left, a Confidence Badge on
the right.

## Overview

The single-row primitive Document Review's Extracted Fields card
repeats for every extracted value. Composes Confidence Badge rather
than reimplementing the percentage-and-chip treatment.

## Live Preview

<Canvas of={ExtractedFieldRowStories.Default} />

## Usage

Stack multiple rows in a card, separated by a divider, to build the
full Extracted Fields list.

## Variants

<Canvas of={ExtractedFieldRowStories.FieldList} />

## Sizes

One size, sized for a ~300px-wide fields sidebar.

## States

Static by default. Pass `editable` to add a Pencil affordance that
turns the value into an inline text field — Enter or blur commits via
`onSave`, Escape reverts. The row doesn't decide what a correction
means (re-running confidence, persisting it, etc.) — that's the
consuming screen's job via `onSave`.

<Canvas of={ExtractedFieldRowStories.Editable} />

## Accessibility

- Label and value are plain text, readable by assistive tech in
  document order.
- Confidence Badge carries its own `aria-label` with the full
  confidence sentence.
- The edit button has an explicit `aria-label` ("Edit {label}") since
  a bare Pencil icon has no accessible name on its own; the inline
  input is associated with the label via a real `<label htmlFor>`.
- The edit affordance is hover-revealed but never `display:none` — it
  reaches full opacity on keyboard focus, so Tab still finds it.
