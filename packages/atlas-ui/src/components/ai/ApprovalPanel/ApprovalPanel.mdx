<DocsLayout>

<DocsTitle>Approval Panel</DocsTitle>

<DocsIntro>
Source: Atlas Product — Document Review's `action-footer`: a summary
line ("7 of 7 fields extracted · 96% avg. confidence") beside Reject /
Request Changes / Approve & Continue actions.
</DocsIntro>

<DocsSection title="Overview">

The bottom action bar for human-in-the-loop approval — a summary of
what was extracted, plus the three actions a reviewer can take.
Composes Button for all three actions.

</DocsSection>

<DocsSection title="Live Preview">

<Canvas of={ApprovalPanelStories.Default} />

</DocsSection>

<DocsSection title="Usage">

Reject and Request Changes are optional — omit `onReject` /
`onRequestChanges` to show only the primary Approve action, for flows
that don't offer those paths.

</DocsSection>

<DocsSection title="Variants">

Tone and layout are fixed to match the grounded Product pattern; the
only variation is which actions are present.

</DocsSection>

<DocsSection title="Sizes">

One size — a full-width footer bar. Wraps (`flex-wrap`) rather than
compressing its buttons on narrow viewports — the summary text takes its
own line first (`min-w-[12rem]` keeps it from squeezing illegibly thin)
before the actions do.

</DocsSection>

<DocsSection title="States">

<Canvas of={ApprovalPanelStories.Loading} />

</DocsSection>

<DocsSection title="Accessibility">

- All three actions are real `<button>` elements via Button, reachable
  and activatable by keyboard.
- `loading` disables every action and shows a busy state on the primary
  button, preventing a double-submit.

</DocsSection>

<DocsSection title="React Props">

<Controls of={ApprovalPanelStories.Default} />

<ArgTypes of={ApprovalPanelStories} />

</DocsSection>

<DocsSection title="Examples">

```tsx
import { ApprovalPanel } from '@/components/ai/ApprovalPanel';

<ApprovalPanel
  summary="7 of 7 fields extracted · 96% avg. confidence"
  onReject={() => rejectDocument()}
  onRequestChanges={() => requestChanges()}
  onApprove={() => approveDocument()}
/>
```
