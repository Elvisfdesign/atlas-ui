# Assistant Panel

Source: Atlas Product — Document Review's `ai-col`: the AI Assistant
sidebar, present alongside a document's viewer and extracted fields.

## Overview

The full AI Assistant experience — a header with a BETA tag, a message
list (intro message plus any conversation), suggested-prompt chips, a
composer, and a disclaimer. Composes AI Label, Suggested Prompt, and
Prompt Input rather than reimplementing any of them.

## Usage

Pass an initial assistant `messages` entry as the greeting, plus
`suggestedPrompts` for the starting chips. Once a conversation begins,
pass the growing `messages` array (alternating `assistant`/`user`
roles) — Assistant Panel doesn't manage conversation state itself.

## Accessibility

- The message list is `role="log"` with `aria-live="polite"` and
  `aria-relevant="additions"` — a reply that arrives after a typing
  delay (the real, asynchronous pattern this panel is built for) is
  announced to screen reader users the same way a sighted user notices
  it appear, without re-announcing the whole transcript on every
  re-render.
- Suggested Prompt and Prompt Input each carry their own accessibility
  behavior (see their docs) unchanged when composed here.

## Examples

```tsx
import { useState } from 'react';
import { AssistantPanel, type AssistantMessage } from '@/components/ai/AssistantPanel';

function DocumentAssistant() {
  const [messages, setMessages] = useState<AssistantMessage[]>([
    { id: '1', role: 'assistant', content: 'Hi, how can I help?' },
  ]);

  const handleSubmit = (value: string) => {
    setMessages((prev) => [...prev, { id: crypto.randomUUID(), role: 'user', content: value }]);
  };

  return (
    <AssistantPanel
      messages={messages}
      suggestedPrompts={['Summarize this document', 'Check for anomalies']}
      onSuggestedPromptClick={handleSubmit}
      onSubmit={handleSubmit}
    />
  );
}
```
