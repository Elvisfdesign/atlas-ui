<DocsLayout>

<DocsTitle>Prompt History Item</DocsTitle>

<DocsIntro>
Not a direct 1:1 Figma frame — Document Review's doc-tabs include a
"History" tab, but its expanded content wasn't part of the captured
screen state. Synthesized from Atlas's established list-item recipe
(File Row / Notification Panel's item button) applied to a past
prompt-and-response pair.
</DocsIntro>

<DocsSection title="Overview">

A single row in a prompt history list — the prompt, an optional
response snippet, and a relative time, as a clickable button.

</DocsSection>

<DocsSection title="Live Preview">

<Canvas of={PromptHistoryItemStories.Default} />

</DocsSection>

<DocsSection title="Usage">

Stack multiple items, separated by a divider, to build a full history
list (e.g. behind Document Review's "History" tab).

</DocsSection>

<DocsSection title="Variants">

<Canvas of={PromptHistoryItemStories.List} />

</DocsSection>

<DocsSection title="Sizes">

One size — a full-width list row.

</DocsSection>

<DocsSection title="States">

Default and hover; a real `<button>`, so keyboard focus and activation
work natively.

</DocsSection>

<DocsSection title="Accessibility">

- Renders as a real `<button>`, reachable and activatable by keyboard —
  same pattern as Notification Panel's item rows.
- The response snippet clamps to two lines visually but remains fully
  readable to assistive tech (no `aria-hidden` truncation).

</DocsSection>

<Controls of={PromptHistoryItemStories.Default} />

<ArgTypes of={PromptHistoryItemStories} />

</DocsSection>

<PromptHistoryItem
  prompt="Summarize this document"
  response="This invoice from Acme Corporation totals $12,450.00, due June 18, 2025."
  time="2m ago"
