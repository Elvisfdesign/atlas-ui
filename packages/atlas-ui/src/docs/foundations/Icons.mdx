# Icons

A single, consistent stroke-based icon set — not a custom-drawn library.

## Overview

Atlas uses [Lucide](https://lucide.dev) directly. There is no Atlas-specific
`<Icon>` wrapper component: components accept a Lucide element as a prop
(see Button's `icon` prop, or Icon Button's `icon` prop), so any of
Lucide's 1000+ icons work without Atlas needing to re-export or re-draw
them one at a time. What follows is a representative sample grouped by the
role icons play in the product — not a catalog of every icon in the set.

## Sizes

Three sizes, matching the Atlas Dimensions scale exactly — never an
arbitrary icon size.

<div style={{ display: 'flex', alignItems: 'flex-end', gap: '32px', margin: '8px 0 32px' }}>
  {[
    { label: 'sm', px: sizing.iconSm },
    { label: 'md', px: sizing.iconMd },
    { label: 'lg', px: sizing.iconLg },
  ].map(({ label, px }) => (
    <div key={label} style={{ display: 'flex', flexDirection: 'column', alignItems: 'center', gap: '8px' }}>
      <Search size={parseInt(px, 10)} color="var(--color-text-primary)" />
      <span style={{ fontFamily: 'Inter', fontSize: 12, color: 'var(--color-text-secondary)' }}>{label}</span>
      <span style={{ fontFamily: 'monospace', fontSize: 11, color: 'var(--color-text-tertiary)' }}>{px}</span>
    </div>
  ))}
</div>

- **sm (14px)** — Small-size Button/Icon Button, dense table rows.
- **md (16px)** — Medium-size Button/Icon Button, the default everywhere else.
- **lg (20px)** — standalone icons, empty-state illustrations.

## Accessibility

- **Decorative icons** (an icon next to a label that already says the same
  thing — e.g. a bell icon beside no text at all in a notifications
  button) should be marked `aria-hidden="true"`. Every icon-accepting Atlas
  component already does this internally — see Icon Button and Button's
  implementations.
- **Meaningful icons with no visible label** (Icon Button) must always be
  paired with an `aria-label` on the interactive element itself — the icon
  is never the only description of what a control does.
- Stroke width: use Lucide's default (`strokeWidth={2}`) unless a specific
  component documents otherwise. Don't mix stroke weights within the same
  view.
