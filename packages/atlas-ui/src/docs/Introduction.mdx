<DocsSection title="Principles">

Five things this library optimizes for, in order:

**Fidelity to Atlas.** This library does not have its own design opinions.
Every color, space, radius, and type size resolves to an Atlas token. Where
Atlas Product and the Atlas UI System disagree slightly, the shipped Product
screens win — they're the ground truth.

**Restraint.** Atlas succeeds because of what it leaves out — no gradients,
no decorative shadows, no variants without a real product need. This
documentation site follows the same rule: it exists to make the system
legible, not to look impressive on its own.

**Accessibility as default, not add-on.** Keyboard support, visible focus
states, and correct ARIA semantics are part of a component's definition, not
a follow-up task. Radix primitives are used wherever hand-rolling
accessibility correctly would mean re-solving a problem that's already
solved (dialogs, tooltips, menus).

**Composition over configuration.** Components expose a small, predictable
prop surface and compose together, rather than growing a prop for every
possible situation.

**Two themes, one component.** Every component reads color through semantic
tokens, so Light and Dark are the same component rendering from different
CSS variables — never a duplicated implementation.

</DocsSection>
